## 2026-06-08 - Information Disclosure via API Errors
**Vulnerability:** The Vercel Edge Functions in the `api/` directory were catching errors from the Anthropic SDK or internal processing and returning raw `e.message` details to the client in HTTP responses (e.g., `Anthropic error: [message]`).
**Learning:** This pattern was consistently used across multiple API endpoints (`api/analyze-clothing.ts`, `api/chat.ts`, `api/suggest-outfit.ts`), indicating a systemic gap in secure error handling practices within this application's architecture. Returning internal error details can expose underlying system behavior, API constraints, or internal paths to potential attackers.
**Prevention:** Always follow the "Fail securely" principle. Ensure that server-side catch blocks log the detailed error using `console.error` for debugging purposes, but return generic, safe, and localized error messages (e.g., "שגיאה פנימית בשרת") to the user. Do not pass `e.message` or stack traces back in HTTP responses.
