## 2026-05-25 - Error Detail Leakage in Vercel Edge Functions\n**Vulnerability:** API Edge functions (`api/chat.ts`, `api/analyze-clothing.ts`, `api/suggest-outfit.ts`) were returning internal error details (`e.message`) directly to the client when exceptions occurred during Anthropic API calls.\n**Learning:** In serverless Edge functions, failing to sanitize error responses can leak sensitive internal details (such as API structures, paths, or potentially API keys) to end users.\n**Prevention:** Always log detailed errors server-side using `console.error` and return generic, non-descriptive error messages (e.g., 'Internal server error') to the client to ensure secure failure.
