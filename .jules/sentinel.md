## 2024-06-16 - Vercel Edge Function Authorization Bypass Risk
**Vulnerability:** The API endpoints (`/api/chat`, `/api/suggest-outfit`, etc.) were completely unauthenticated, allowing anyone to bypass the frontend and directly interact with the backend Edge Functions (consuming Anthropic API quota and potentially accessing functionalities unauthorized).
**Learning:** In hybrid "local-first/cloud-sync" architectures where an application uses localStorage when Supabase is unconfigured, API routes often fail to enforce authorization for logged-in users. Edge Functions mapped under Vercel cannot rely on frontend context and must explicitly validate authentication tokens themselves.
**Prevention:** Always extract and validate the `Authorization: Bearer <token>` header within the Edge Function handler using `supabase.auth.getUser(token)` before performing any sensitive operations or external API calls, conditionally checking if the environment requires it.
